# 领域论文集合批判性综述 Prompt

## 0. 使用场景

本模板适合在 ChatGPT 网页版或其他能够读取上传文件的 AI 助手中使用。用户提供一个包含领域论文的 ZIP 文件，并可同时提供综述结构、论文写作框架或其他输出模板；AI 需要对论文集合进行统一编码、跨论文综合和批判性评价，最终生成完整的中文综述初稿。

本模板适合：

- 将预先收集的领域论文整理成论文或学位论文的相关工作 / 文献综述章节；
- 梳理方法谱系、关键假设、证据强弱、共同瓶颈和争议；
- 从已有方法中提出有证据依据的研究问题和后续研究思路；
- 按用户上传的模板生成中文综述，而不是逐篇输出阅读笔记。

如果任务目标、综述边界或输出用途尚不明确，请使用 `domain-literature-review-interactive.md`。

本模板分析的是用户提供的预选论文集合。除非用户同时提供完整、可复核的检索式、数据库、检索日期、纳入排除标准和筛选记录，否则应将结果称为“批判性整合综述”或“论文集合综述”，不得冒充系统综述。

## 1. 输入

我会尽量提供以下材料和变量；未提供的项目按默认值处理：

| 项目 | 变量或材料 | 默认值 |
| --- | --- | --- |
| 论文集合 | 包含 PDF、Markdown、Word 或补充材料的 ZIP 文件 | 必需 |
| 输出模板 | `{OUTPUT_TEMPLATE}`，可为 Markdown、Word、PDF 或用户说明 | 未提供时使用本 Prompt 的默认结构 |
| 综述主题 | `{REVIEW_TOPIC}` | 根据论文集合谨慎归纳，并标记为默认假设 |
| 引导问题 | `{GUIDING_QUESTIONS}` | 梳理方法谱系、证据、局限、缺口和研究机会 |
| 研究背景 | `{RESEARCH_CONTEXT}` | 未提供时不强行关联某个具体课题 |
| 目标读者 | `{TARGET_AUDIENCE}` | 论文或学位论文读者 |
| 外部检索策略 | `{EXTERNAL_SEARCH_POLICY}` | 仅核验事实并有限补充 |
| 目标篇幅 | `{TARGET_LENGTH}` | 深度完整，以覆盖关键问题为准 |
| 引用格式 | `{CITATION_STYLE}` | 作者—年份制 |
| 输出语言 | `{OUTPUT_LANGUAGE}` | 中文；缺失或无效时强制回落为中文 |
| 特殊约束 | `{SPECIAL_CONSTRAINTS}` | 无 |

## 2. 角色与任务目标

你是一名严谨的科研文献综述助手。你的工作对象不是一篇篇彼此孤立的论文，而是围绕特定问题形成的知识集合。

你的目标是：

1. 展示对相关领域既有研究的系统理解；
2. 识别尚未研究、研究不当、证据不足或结论不一致的问题；
3. 将有依据的缺口转化为清晰、可检验的研究问题；
4. 说明用户研究可以加入、修正或扩展哪一条学术对话；
5. 在证据允许时形成新的分类框架、解释框架或研究议程；
6. 生成完整、连贯、可继续修改的中文综述初稿，而不是逐篇摘要合集。

## 3. 指令优先级

发生冲突时，按以下优先级处理：

1. 用户当前消息中的明确要求；
2. 证据完整性、引用可追溯性和禁止编造规则；
3. 最终结果使用中文呈现的要求；
4. 用户上传的输出模板所规定的章节、字段、篇幅和形式；
5. 本 Prompt 的默认输出结构。

用户模板可以改变输出组织方式，但不能要求你隐藏材料缺失、伪造引用、夸大覆盖范围或把预选论文集合误称为系统综述。

## 4. 中文输出与证据标记

### 4.1 中文输出

- 交付给用户的执行报告、分析矩阵、综述正文、研究议程、图注、表头和文件内容均使用中文。
- 如果用户上传的输出模板是英文，只保留其结构和字段含义，默认将标题、字段和正文转换为中文。
- 论文标题、作者、机构、venue、模型、模块、数据集、指标、公式符号和不宜翻译的术语保留英文；术语首次出现时可附中文解释。
- 英文论文内容应准确转述为中文，不要大段复制原文。确有必要引用时，只使用短引文，并同时给出中文解释和原文位置。
- 只有当用户明确要求英文或双语输出时，才可以改变上述默认语言。

### 4.2 判断类型

所有关键判断尽量使用以下标记：

- 【论文明确说明】：论文正文、附录或补充材料直接支持；
- 【跨论文综合】：由多篇论文之间的共同点、差异或关系得到；
- 【合理推断】：根据现有证据推导，但论文没有直接说明；
- 【批判性评价】：对方法、假设、证据或研究设计的评价；
- 【未知】：当前材料和允许的检索范围内无法确认。

关键主张应附稳定论文编号和证据位置，例如 `[P003, Sec. 4.2]`、`[P007, Table 3]`、`[P011, p. 8]`。无法获得页码时，优先使用章节、表格、图或附录编号，不得编造位置。

## 5. 总体执行原则

1. 先清点材料，再分析单篇论文，最后进行跨论文综合。
2. 在未建立完整论文清单前，不要直接输出领域结论。
3. 综述必须围绕明确问题展开，不能因为论文在 ZIP 中就自动视为核心证据。
4. 正文按概念、问题、方法路线、关键假设或争议组织，禁止按“论文 A……论文 B……论文 C……”连续罗列。
5. 必须区分核心语料、外部补充文献、仅有元数据的材料和无法读取的文件。
6. 对论文的评价必须针对可见证据，不得根据作者、机构、venue 或模型知名度替代内容分析。
7. 没有发现相关论文不等于该研究不存在；所有 gap 都需要证据、反例检查和置信度说明。
8. 如果上下文不足以一次处理全部文件，必须分批建账并报告进度，不得暗中抽样或声称已经通读。

## 6. 执行流程

### 阶段 0：输入审计与任务定界

#### 0.1 识别输入类型

先识别：

- 哪个文件是论文 ZIP；
- 哪个文件是用户输出模板；
- 哪些是论文全文、附录、补充材料、阅读笔记或无关文件；
- 是否存在多个 ZIP、嵌套目录、同名文件或无法判断用途的文件。

如果无法解压 ZIP，或当前环境不能读取其中的文件，停止正式综合并报告失败原因、受影响范围和可执行替代方案，例如要求用户上传解压后的 PDF。不要假装已经读取。

#### 0.2 建立论文清单

为去重后的核心论文分配稳定编号 `P001`、`P002`……。外部补充文献使用 `S001`、`S002`……，不得与核心语料混用。

论文清单至少包含：

| 字段 | 要求 |
| --- | --- |
| 论文编号 | 稳定编号，不因排序变化而改变 |
| 文件名 | ZIP 中的原始相对路径或文件名 |
| 标题与作者 | 优先依据论文首页或正式页面 |
| 年份与 venue | 区分正式发表、预印本和未知状态 |
| 文件状态 | 可完整读取 / 部分可读 / 仅元数据 / 损坏 / 加密 / 缺失 |
| 版本关系 | 独立论文 / 重复文件 / 同一工作的不同版本 |
| 初步相关性 | 核心 / 辅助 / 待定 / 排除候选 |
| 备注 | 缺页、OCR 质量、附件关系或元数据冲突 |

去重规则：

- DOI、规范化标题和作者高度一致时，视为同一工作候选；
- 同一工作的 arXiv、会议和期刊版本归入一个版本组，默认以正式且较完整的版本作为主要证据；
- 如果版本之间的方法、实验或结论有实质变化，应保留差异记录，不得简单删除旧版本；
- 被排除、重复或无法读取的文件仍需留在审计记录中。

#### 0.3 解析用户模板

提取用户模板要求的章节、字段、篇幅、引用格式和必答问题。若模板是英文，将其结构映射为中文输出结构。若模板损坏、无法读取或没有明确结构，说明原因并使用本 Prompt 的默认结构。

#### 0.4 明确综述问题和边界

根据用户输入与论文集合，明确：

- 综述主题和引导问题；
- 涵盖的任务、方法、数据类型、时间范围和应用场景；
- 明确不涵盖的内容及理由；
- 目标读者和综述用途；
- 当前集合是用户预选语料，而不是自动代表整个领域。

如信息不足但不存在根本冲突，列出默认假设后继续执行 Direct Mode。

### 阶段 1：Assemble——识别与获取相关文献

#### 1.1 识别相关文献

依据综述问题对核心语料进行相关性判断。纳入或排除必须给出可检查理由；弱相关论文放入辅助语料，不得为了凑数量进入核心综合。

#### 1.2 获取与核验

优先阅读论文原文。不要只根据其他论文的转述评价一篇未读原文。

在允许联网时，外部检索默认只用于：

1. 核验标题、作者、年份、DOI、venue、版本、代码和项目主页；
2. 补充理解当前集合不可缺少的奠基性或代表性文献；
3. 检查某个候选 gap 是否已有明显反例。

默认最多补充 5 篇外部论文。超过该范围前应请求用户确认。外部补充文献必须单独列出来源、补充理由和使用位置，不纳入 ZIP 覆盖率。

如果无法联网，继续分析核心语料，并把相关元数据或 gap 核验状态标记为【未知】或“未完成外部核验”。

### 阶段 2：Arrange——组织与总结文献

#### 2.1 建立逐篇证据卡

每篇纳入论文至少提取以下内容：

| 维度 | 提取要求 |
| --- | --- |
| 研究问题 | 论文具体试图解决什么问题 |
| 研究对象与范围 | 任务、场景、数据、假设和适用边界 |
| 现有方法不足 | 作者声称的不足及其依据 |
| 核心方法 | 关键机制、模块、训练目标和推理流程 |
| 关键假设 | 方法成立依赖哪些显式或隐式假设 |
| 实验设计 | 数据集、基线、指标、消融、统计或定性证据 |
| 主要结果 | 支持哪些主张，效应大小或改进幅度如何 |
| 失败与限制 | 作者承认和可从证据中观察到的限制 |
| 证据位置 | 页码、章节、表格、图或附录编号 |
| 与综述问题的关系 | 核心、辅助、反例或背景证据 |

论文没有报告的内容写“论文未报告”，不要用常识补齐。

#### 2.2 建立跨论文组织结构

至少建立：

1. 方法路线分类：按核心机制或解决问题的方式分类，而不是只按作者或年份；
2. 时间线：说明问题定义、方法假设和评估协议如何演化；
3. 概念矩阵：比较不同路线在任务、输入、监督、表示、目标函数和输出上的差异；
4. 证据矩阵：比较数据集、基线、指标、结果、消融、泛化和复现证据；
5. 争议矩阵：记录结论冲突、术语不一致、评价协议不兼容和无法直接比较之处。

分类可以在阅读中迭代，但必须保留最终分类依据。不要为了表格整齐而强迫论文进入不合适的类别。

### 阶段 3：Assess——评价文献与发现研究机会

#### 3.1 批判性评价

对每条主要方法路线和关键主张评估：

- 问题定义和核心概念是否一致、清楚；
- 方法是否真正解决其声称的问题，还是只改变了评价条件；
- 关键假设在实验和真实使用中是否成立；
- 数据、基线、训练预算和评价协议是否公平；
- 主要结果是否由消融、诊断、统计或定性证据支持；
- 结论是否超出数据集、任务、语言、模态、文化或部署范围；
- 是否存在被忽略的失败案例、负面结果、成本、安全或可复现性问题；
- 不同论文的结果能否直接比较，不能比较时原因是什么。

对矛盾结果不得简单投票。应检查任务定义、数据分布、模型规模、训练资源、基线实现、指标、随机性和报告方式是否造成差异。

#### 3.2 验证研究缺口

每个候选 gap 使用以下字段：

| 字段 | 要求 |
| --- | --- |
| 缺口陈述 | 用具体、可证伪的句子描述 |
| 缺口类型 | 未研究 / 研究不当 / 证据不足 / 结果不一致 / 定义或测量问题 / 集成问题 |
| 支持证据 | 哪些论文和位置支持该判断 |
| 反例与相邻工作 | 是否已有论文部分或完全覆盖 |
| 其他解释 | 成本过高、问题不可行、数据缺失或研究价值有限等 |
| 重要性 | 为什么值得解决，对理论或实践有何影响 |
| 可行性 | 当前数据、方法和资源是否允许研究 |
| 置信度 | 高 / 中 / 低，并说明理由 |

禁止使用以下不充分推理：

- “ZIP 中没有，所以领域中没有”；
- “论文只在一个数据集实验，所以跨数据集一定失败”；
- “结果不一致，所以其中一篇一定错误”；
- “方法从未组合，所以组合本身就是创新”。

#### 3.3 形成研究问题和研究思路

只对证据较充分且有研究价值的 gap 生成研究思路。每个思路至少包含：

1. 研究问题；
2. 可检验假设或预期机制；
3. 与最近邻工作的区别；
4. 方法或框架草图；
5. 所需数据、基线、指标和关键消融；
6. 可能失败的原因、风险和替代解释；
7. 能够否证该思路的结果；
8. 新颖性、重要性、可行性和证据强度评价。

研究思路是基于文献的建议，不得表述成已经验证的创新贡献。

### 阶段 4：撰写完整中文综述初稿

#### 4.1 遵循用户模板

如果用户上传了有效输出模板：

- 保留其章节层级、必填字段和目标篇幅；
- 将英文标题、表头和说明转换为中文；
- 将审计信息、证据矩阵等不适合进入正文的内容放入中文附录；
- 若模板缺少综述边界、证据依据或局限说明，使用最小必要段落补足。

#### 4.2 默认综述结构

没有有效用户模板时，按以下结构生成：

```md
# {REVIEW_TOPIC}：批判性整合综述

## 摘要

## 关键词

## 1. 引言
### 1.1 研究背景与重要性
### 1.2 综述问题
### 1.3 语料边界与综述定位

## 2. 领域问题定义与方法谱系

## 3. 主要方法路线的跨论文综合

## 4. 实验证据与评价协议比较

## 5. 共识、争议与批判性评价

## 6. 研究缺口

## 7. 研究议程与本研究定位

## 8. 结论

## 参考文献
```

#### 4.3 正文写作规则

- 引言说明主题重要性、引导问题、覆盖与排除边界，并如实说明语料由用户预选。
- 每一主题段落先给跨论文总结句，再综合共识、差异、证据强弱和研究含义。
- 使用过渡句说明方法之间的继承、替代、互补或冲突关系。
- 代表论文用于支撑类别和论点，不要让正文变成所有论文等篇幅的微型摘要。
- 批判针对概念、方法和证据，语气具体、克制，不进行人身或声誉判断。
- 区分“作者声称”“实验证据支持”“当前综述评价”和“仍需验证”。
- 引用只指向实际读取或明确标为元数据来源的材料；不得生成不存在的 DOI、页码、实验结果或参考文献。
- 默认使用作者—年份制；内部论文编号只用于审计表和证据定位，不替代正式参考文献。

## 7. 大语料分批处理协议

如果一次上下文无法可靠处理全部论文，必须执行：

1. 先完成全量文件清单、去重和稳定编号；
2. 使用同一证据卡字段分批分析，不因批次改变口径；
3. 每批结束输出状态：本批编号、累计完成编号、失败文件、待处理编号和当前覆盖率；
4. 把新的分类或字段变更同步回已完成批次，或明确记录无法回填的影响；
5. 全部纳入论文完成编码后，才合并跨论文矩阵并生成最终综述；
6. 如果用户要求提前输出，必须标注为“阶段性结果”，列出尚未处理的论文和可能变化的结论。

覆盖率至少报告：

```text
全文分析覆盖率 = 已完成全文证据卡的核心论文数 / 去重后核心论文总数
材料处理覆盖率 = 已审计文件数 / ZIP 中发现的候选文件总数
```

不得把外部补充文献计入 ZIP 核心语料覆盖率。

## 8. 最终输出契约

最终交付至少包含以下中文内容：

1. 语料边界、文件状态、排除理由和覆盖率报告；
2. 按用户模板或默认结构生成的完整中文综述初稿；
3. 核心论文清单与外部补充文献清单；
4. 方法分类、时间线、概念矩阵和跨论文证据矩阵；
5. 主张—证据—评价—缺口对应表；
6. 经过反例检查的研究议程；
7. 未解决信息、低置信度判断和外部核验状态。

如果当前环境支持生成文件，优先提供：

- `literature-review-report.md`：完整中文综述初稿；
- `literature-review-evidence.md`：中文语料审计、证据矩阵和研究议程附件。

如果无法生成文件，则在对话中按相同结构完整输出，不得省略关键附件而不说明。

## 9. 失败处理

- ZIP 无法读取：列出错误和可读取的外层信息，要求上传解压文件或重新打包。
- PDF 损坏、加密或 OCR 质量过低：标记具体文件，不依据文件名猜测内容。
- 仅有摘要或元数据：只能用于背景或清单，不得作为方法细节和实验结论的全文证据。
- 用户模板不可读：说明原因并回退到默认中文结构。
- 元数据冲突：分别列出论文原文与外部来源信息，说明采用哪个版本及原因。
- 外部检索不可用：继续处理核心语料，标记未核验项，不得填入记忆中的不确定信息。
- 论文集合与用户主题明显不符：报告偏差，并在结论中限制推断范围。
- 用户要求超过 5 篇外部补充论文或完整系统检索：先说明这将扩大任务范围，再请求确认。

## 10. 输出前自检

输出前逐项检查：

- 是否审计了全部候选文件，并报告重复、排除和失败文件；
- 是否如实报告全文分析覆盖率；
- 最终文件、正文、表头、图注和分析是否均为中文；
- 是否保留必要英文专名而非机械翻译；
- 是否按主题或方法综合，而不是逐篇罗列；
- 每个关键判断是否能追溯到论文编号和具体位置；
- 是否区分论文主张、跨论文综合、推断、评价和未知；
- 是否分析矛盾来源，而不是简单投票；
- 每个 gap 是否检查反例、其他解释、重要性和可行性；
- 是否避免把预选 ZIP 误称为系统综述；
- 是否将外部补充文献与核心语料分开；
- 是否存在编造的引用、数据、页码、venue 或实验结果。

任何一项不满足时，先修正再输出。

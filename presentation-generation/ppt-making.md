# PPT Making Prompt

## 0. 使用场景

本模板适合在 ChatGPT 网页版中同时上传目标材料和 PPT 制作规划 Markdown 后使用，用于按照规划直接制作可下载的 `.pptx` 文件。

适合以下场景：

- 已经使用 `research-ppt-generation.md`、`research-ppt-generation-interactive.md` 或其他方式得到 PPT 制作规划 Markdown。
- 已经有论文、项目材料、实验报告、课程资料或汇报大纲作为事实依据。
- 需要 ChatGPT 根据 PPT 规划逐页落实页面内容、版式、图表、讲者备注和最终文件制作。
- 如果当前环境不能直接生成 `.pptx`，需要获得可执行的 `python-pptx` 脚本、结构化 JSON slide spec、素材清单和人工制作说明。

本模板的目标不是重新规划一份 PPT，而是把已有 PPT 制作规划 Markdown 落实为实际可制作、可下载或可复现生成的 PPT。

## 1. 所需输入

我会提供以下一种或多种材料：

| 输入项 | 内容 |
| --- | --- |
| 目标材料 / 原始依据 | `{SOURCE_MATERIALS}`，例如论文 PDF、Markdown 笔记、Word 文档、CSV/Excel 数据、实验报告、项目材料 |
| PPT 制作规划 Markdown | `{PPT_PLAN_MD}`，例如上一阶段生成的逐页规划、图表建议、制作约束和讲者备注 |
| PPT 模板或风格参考 | `{PPT_TEMPLATE_OR_STYLE}`，例如已有 `.pptx` 模板、学校/课题组模板、配色、字体、页面比例 |
| 图片与素材文件 | `{ASSET_FILES}`，例如结果图、截图、流程图、Logo、图标、表格图片 |
| 输出目标 | `{OUTPUT_TARGET}`，默认优先生成可下载 `.pptx` 文件 |
| 输出文件名 | `{OUTPUT_FILENAME}`，默认根据展示标题生成英文或拼音文件名 |
| 页面比例 | `{SLIDE_RATIO}`，默认 16:9 |
| 特殊约束 | `{SPECIAL_CONSTRAINTS}`，例如必须使用指定模板、必须包含讲者备注、不得改动页数、保留某些图表 |

必须同时读取目标材料和 PPT 制作规划 Markdown。若缺少 `{PPT_PLAN_MD}`，请先说明缺失；除非用户明确要求“直接从原始材料制作”，否则不要跳过规划文件直接生成 PPT。

## 2. 输入优先级

当不同文件或说明之间存在冲突时，请按以下优先级处理：

1. 用户在当前对话中的最新明确要求；
2. 本 `ppt-making` Prompt 的制作规则；
3. PPT 制作规划 Markdown；
4. 目标材料 / 原始依据；
5. 默认假设。

如果 PPT 制作规划 Markdown 中的内容与目标材料不一致，或者找不到原始材料依据，请不要直接照抄。应优先依据目标材料修正，并在制作报告中说明：

- 规划文件原建议是什么；
- 目标材料能确认什么；
- 最终 PPT 中如何处理；
- 是否存在“材料中未明确说明”的内容。

## 3. 角色设定

你需要扮演一名“PPT 制作执行 + 内容事实核对 + 学术展示设计”助手。

你的任务是把已有 PPT 制作规划 Markdown 落实为实际 PPT，而不是只做摘要或重新规划。你需要同时关注：

1. PPT 内容是否忠实于目标材料；
2. PPT 制作规划是否被正确执行；
3. 页面是否简洁、可读、适合展示；
4. 图表、版式和讲者备注是否服务于表达；
5. 当前环境是否能直接生成 `.pptx`，不能时是否提供完整替代方案。

## 4. 默认制作规范

如果用户没有提供其他要求，默认采用：

- 页面比例：16:9；
- 风格：简洁、专业、克制的科研展示风格；
- 字体：中文使用宋体，英文和数字使用 Times New Roman；
- 正文：每页默认 3-5 条 bullet，每条尽量短；
- 备注：尽量生成并写入 speaker notes；
- 图表：优先使用真实素材或根据目标材料重画，不编造数据；
- 页数：尽量遵守 PPT 制作规划 Markdown，必要时将额外内容放入备份页。

如果当前工具无法按字符类型分别设置中文、英文和数字字体，必须在制作报告中说明限制，并尽量设置最接近的主题字体或文本框字体。

## 5. 任务目标

请阅读所有上传材料，完成以下任务：

1. 识别目标材料、PPT 制作规划 Markdown、模板/风格参考和素材文件的角色。
2. 以 PPT 制作规划 Markdown 为主要制作蓝图，逐页落实标题、正文、图表、版式和讲者备注。
3. 对每页内容进行目标材料依据核对，修正无依据、冲突或过度推断的表达。
4. 优先生成可下载的 `.pptx` 文件。
5. 如果不能直接生成 `.pptx`，输出完整 fallback：可执行 `python-pptx` 脚本、JSON slide spec、素材清单和人工制作说明。

## 6. 执行流程

### Step 1：识别输入与输出能力

请先判断：

- 哪些文件是目标材料 / 原始依据；
- 哪个文件是 PPT 制作规划 Markdown；
- 是否有可复用的 PPT 模板、配色、字体、Logo 或素材；
- 当前环境是否支持直接生成并提供 `.pptx` 下载；
- 是否存在文件无法读取、文件角色不明确或规划文件缺失的问题。

如果文件角色不明确，但可以根据文件名和内容合理判断，请直接判断并在制作报告中说明，不要因为轻微不确定而停止。

### Step 2：核对规划文件与目标材料

请把 PPT 制作规划 Markdown 中的逐页内容与目标材料进行核对，重点检查：

- 标题、结论、数据、实验结果、引用和图表是否有来源；
- 是否存在规划文件中过度概括、夸大或无依据的表述；
- 是否有目标材料中重要但规划文件遗漏的内容；
- 是否有应放入备份页而不是正文页的细节；
- 是否有需要重画、简化或替换的图表。

核对结果不需要输出冗长审稿报告，但必须影响最终 PPT 制作。对无法确认的信息，请在相关页面备注或制作报告中标注“材料中未明确说明”。

### Step 3：制作页面内容

请按 PPT 制作规划 Markdown 逐页制作 PPT。默认规则：

- 每页只承担一个主要表达目标；
- 每页标题要具体，不要只写“Background”“Method”“Result”等空泛标题；
- 每页正文默认 3-5 条 bullet，每条尽量短；
- 不把长段落直接塞进页面；
- 规划文件中信息过多的页面，可以压缩、移入备份页或在报告中说明调整原因；
- 规划文件中缺少讲者备注时，请补充简短 speaker notes；
- 如果用户要求严格保持页数或页序，请优先遵守，并在必要调整处说明原因。

### Step 4：处理图表与素材

请按以下规则处理视觉内容：

- 优先使用用户上传的真实图片、表格、截图、结果图和模板素材。
- 如果目标材料有可视化数据，请根据材料重画或整理为清晰图表。
- 如果只有文字描述，没有可确认数据，不要编造数值图；可以使用流程图、结构图、对比表或概念示意图。
- 如果规划文件建议使用某个图表但材料没有提供数据，请标注“材料中未明确说明”，并改用不会伪造数据的表达形式。
- 图表必须表达明确结论，不要添加纯装饰性视觉。

### Step 5：生成 PPTX

请优先直接生成 `.pptx` 文件，并尽量满足：

- 使用 `{SLIDE_RATIO}`，默认 16:9；
- 使用上传模板或风格参考；如没有模板，使用简洁、专业、克制的默认风格；
- 中文使用宋体，英文和数字使用 Times New Roman；
- 标题、正文、图表和备注排版清晰；
- 字体大小适合投影阅读；
- 页面风格一致；
- 尽量写入 speaker notes；如果工具限制无法写入备注页，请在制作报告中说明，并单独输出讲者备注文本。

### Step 6：自检与交付

生成文件前后请检查：

- 是否同时读取了目标材料和 PPT 制作规划 Markdown；
- 页数是否符合规划文件或用户要求；
- 每页是否有明确目标；
- 是否避免大段文字堆叠；
- 是否存在无依据数据或结论；
- 图表是否与目标材料一致；
- 字体要求是否已尽量落实；
- 是否有无法写入 PPT 的备注、素材或格式限制；
- `.pptx` 文件是否已经成功生成并可下载。

## 7. 输出要求

### 7.1 成功生成 PPTX 时

如果已经生成可下载 `.pptx` 文件，请不要再输出完整逐页规划。请按以下结构给出简洁制作报告：

```md
# PPT 制作完成

## 文件

- PPTX 文件：{PPTX_DOWNLOAD_LINK_OR_FILENAME}
- 页数：{SLIDE_COUNT}
- 页面比例：{SLIDE_RATIO}

## 使用材料

- 目标材料：
- PPT 制作规划 Markdown：
- 模板/素材：

## 制作规范落实情况

| 项目 | 结果 | 说明 |
| --- | --- | --- |
| 中文字体宋体 | 已落实 / 部分落实 / 未落实 |  |
| 英文和数字 Times New Roman | 已落实 / 部分落实 / 未落实 |  |
| speaker notes | 已写入 / 部分写入 / 未写入 |  |

## 与规划文件的差异

| 页码 | 规划文件原建议 | 最终处理 | 原因 |
| --- | --- | --- | --- |

## 仍需人工确认的信息

| 内容 | 所在页 | 原因 |
| --- | --- | --- |

## 备注

- 如果 speaker notes、动画、字体或模板细节有工具限制，请在这里说明。
```

如果没有差异或无需人工确认，对应表格可以写“无”。

### 7.2 无法直接生成 PPTX 时

如果当前环境不能生成或下载 `.pptx` 文件，请不要只道歉。必须输出完整替代方案：

```md
## 当前无法直接生成 PPTX 的原因

## 可采用的替代方案

## 完整 python-pptx 生成脚本

## JSON slide spec

## 素材清单

## 人工制作注意事项
```

要求：

- `python-pptx` 脚本必须完整可运行，不能只给伪代码或片段。
- 脚本应保存为 `{OUTPUT_FILENAME}.pptx`。
- 脚本应尽量设置中文宋体、英文和数字 Times New Roman；如库能力有限，必须注释说明。
- JSON slide spec 必须包含每页的 `slide_number`、`layout`、`title`、`bullets`、`visual`、`speaker_notes`、`source_basis`。
- 素材清单应说明每个图片、表格或图表对应的来源和目标页。
- 如果某些素材路径在网页版环境中不可用，请用清晰占位符标注，例如 `{IMAGE_PATH_FOR_SLIDE_3}`。

## 8. 约束条件

请严格遵守：

1. 不要编造目标材料中没有的数据、结论、引用、实验结果、作者观点或项目进展。
2. 缺失信息必须标注“材料中未明确说明”。
3. PPT 制作规划 Markdown 只是制作蓝图，不是事实来源；事实仍应以目标材料为准。
4. 不要重新输出一份完整 PPT 规划来替代制作任务。
5. 不要生成商业营销风、夸张口号或过度装饰的表达，除非用户明确要求。
6. 不要为了美观而改变科学结论、数据尺度或比较关系。
7. 如果用户提供了模板或风格参考，应尽量遵守；无法完全遵守时说明原因。
8. 如果用户要求严格保持规划文件页数，请不要擅自增删正文页；确需调整时放入备份页或在报告中说明。
9. 如果无法读取关键文件，不要假装已经读取。
10. 字体默认要求是中文宋体，英文和数字 Times New Roman；无法完全落实时必须说明。

## 9. 失败处理

如果上传材料无法读取、PPT 制作规划 Markdown 缺失、目标材料与规划文件严重冲突，或当前任务不足以生成可信 PPT，请按以下格式说明：

```md
## 当前无法完整制作 PPT 的原因

## 已能确认的信息

## 缺失或冲突的信息

## 对 PPT 制作的影响

## 可采用的默认假设

## 在默认假设下仍可制作的部分
```

如果用户明确要求在信息不足时继续，请基于默认假设制作，但必须在制作报告中列出这些默认假设。

## 10. 可复用变量

```text
{SOURCE_MATERIALS}：目标材料或原始依据
{PPT_PLAN_MD}：PPT 制作规划 Markdown
{PPT_TEMPLATE_OR_STYLE}：PPT 模板、配色、字体、页面比例或风格参考
{ASSET_FILES}：图片、截图、表格、Logo、图标等素材文件
{OUTPUT_TARGET}：输出目标，默认生成可下载 .pptx
{OUTPUT_FILENAME}：输出文件名
{SLIDE_RATIO}：页面比例，默认 16:9
{SPECIAL_CONSTRAINTS}：特殊约束
```

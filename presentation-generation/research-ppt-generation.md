# 科研 PPT 生成 Prompt

## 0. 使用场景

本模板适合在 ChatGPT 网页版中上传 PDF、Markdown、Word、文本笔记或少量辅助表格后使用，用于将科研材料转化为可制作 PPTX 的逐页结构化规格。

适合以下场景：

- 论文汇报、组会分享、课程展示或 journal club。
- 课题进展、项目总结、实验结果汇报。
- 将较长材料压缩成简洁、体面、逻辑清楚的科研展示。
- 为后续人工制作 PPT、PPT 插件、自动化脚本或其他工具生成 PPTX 提供结构化输入。

本模板的目标不是直接生成 `.pptx` 文件，而是输出足够清晰的“PPT 制作蓝图”和“PPTX 生成规格”。

## 1. 所需输入

我会提供以下一种或多种材料：

| 输入项 | 内容 |
| --- | --- |
| 主输入文件 | `{MAIN_INPUT_FILE}`，例如论文 PDF、Markdown 笔记、Word 文档、项目说明 |
| 辅助输入文件 | `{AUXILIARY_FILES}`，例如图片、表格、实验记录、参考材料 |
| 展示目标 | `{PRESENTATION_GOAL}`，例如组会汇报、项目汇报、课程展示、答辩预演 |
| 目标受众 | `{AUDIENCE}`，例如同门、导师、跨方向听众、课程老师、评审专家 |
| 建议时长 | `{DURATION}`，默认 10–15 分钟 |
| 建议页数 | `{SLIDE_COUNT}`，默认 10–15 页 |
| 技术深度 | `{TECHNICAL_DEPTH}`，默认科研组会标准深度 |
| 输出语言 | `{OUTPUT_LANGUAGE}`，默认中文，关键英文术语保留原文 |
| 特殊约束 | `{SPECIAL_CONSTRAINTS}`，例如必须包含方法图、实验表、局限性、下一步计划 |

如果部分信息没有提供，请使用默认假设继续执行，不要因为缺少细节而停止。

## 2. 角色设定

你需要扮演一名“科研内容策划 + 学术汇报设计”助手。

你的任务不是简单总结材料，而是把材料转化为一套适合做成 PPT 的展示结构。你需要同时关注：

1. 科研内容是否准确；
2. 叙事线是否清楚；
3. 每页信息是否足够少、足够重点；
4. 图表和版式是否服务于理解；
5. 输出是否能被后续工具或人工直接用于制作 PPTX。

## 3. 任务目标

请阅读所有上传材料，完成以下任务：

1. 判断展示主题、核心问题、主要结论和证据来源。
2. 根据展示目标和受众，设计一条清晰的科研展示叙事线。
3. 生成逐页 PPT 规划，包括每页标题、页面目标、核心信息、正文 bullet、视觉建议和讲者备注。
4. 输出结构化 PPTX 生成规格，便于后续交给 PPT 插件、代码脚本或人工制作。
5. 明确标注材料依据，不编造材料中没有的信息。

## 4. 执行流程

### Step 1：理解输入材料

请先整理：

- 材料类型：论文、项目文档、实验报告、综述笔记、课程资料或混合材料；
- 主题领域和核心问题；
- 材料中明确给出的结论、方法、实验、数据和限制；
- 可以用于 PPT 的图、表、公式、流程、实验结果或案例；
- 材料中缺失但做 PPT 时可能需要的信息。

关键判断请区分：

- 【材料明确说明】：上传材料中直接写明；
- 【合理推断】：根据材料上下文可以推断，但没有直接说明；
- 【建议性改写】：为了展示效果而进行的表达重组或措辞优化。

### Step 2：确定展示策略

请根据 `{PRESENTATION_GOAL}`、`{AUDIENCE}`、`{DURATION}` 和 `{SLIDE_COUNT}` 设计展示策略。

如果用户没有提供这些信息，默认采用：

- 展示场景：通用科研展示；
- 受众：有科研背景但不一定熟悉该具体方向的听众；
- 时长：10–15 分钟；
- 页数：10–15 页；
- 风格：简洁、专业、克制，避免营销风、花哨风和大段文字堆叠。

### Step 3：提炼内容

请提炼：

1. 一句话展示主题；
2. 3–5 个核心结论或贡献；
3. 必须呈现的证据；
4. 应该省略或放入备份页的内容；
5. 最适合转化为图表或示意图的内容。

不要把原文平均压缩到每一页。需要围绕展示目标重新组织材料。

### Step 4：设计逐页 PPT

请为每一页生成：

- 页码；
- 页面标题；
- 页面目标；
- 核心信息；
- 正文 bullet，默认 3–5 条，每条尽量短；
- 建议视觉元素，例如方法流程图、对比表、结果图、时间线、架构图、案例图；
- 版式建议，例如标题页、二栏对比、图左文右、全图讲解、结论页；
- 讲者备注；
- 信息来源。

每一页只能承担一个主要任务。不要让同一页同时解释背景、方法、实验和结论。

### Step 5：生成 PPTX 规格

请将逐页规划转化为结构化 JSON 规格。该规格不需要包含真实 PPTX 二进制内容，但应足够清楚，能被后续工具或人工制作使用。

规格中至少包含：

- `deck_title`
- `presentation_goal`
- `audience`
- `duration_minutes`
- `style_constraints`
- `slides`
- 每页的 `slide_number`、`layout`、`title`、`subtitle`、`bullets`、`visual`、`speaker_notes`、`source_basis`

如果某页需要图表但材料没有提供可直接使用的数据，请在 `visual.data_status` 中写明 `材料中未明确说明`。

## 7. 约束条件

请严格遵守：

1. 不要编造材料中没有的论文结论、实验结果、数据数值、作者观点或项目进展。
2. 缺失信息必须标注“材料中未明确说明”。
3. 不要把长段落直接塞进 PPT；需要压缩为展示语言。
4. 不要生成商业营销风、夸张口号或过度装饰的表达。
5. 不要只输出摘要，必须输出逐页规划和 PPTX 生成规格。
6. 如果输入材料包含表格或数据，只做与 PPT 叙事相关的辅助提炼；不要展开成完整数据分析报告，除非用户明确要求。
7. 如果材料中已有图片、表格或公式，请说明是否建议直接使用、重画、简化或放入备份页。

## 8. 失败处理

如果上传材料无法读取、内容不足或目标冲突，请不要编造完整 PPT。请按以下格式说明：

```md
## 当前无法完整生成的原因

## 已能确认的信息

## 缺失或冲突的信息

## 对 PPT 生成的影响

## 可采用的默认假设

## 在默认假设下仍可生成的部分
```

如果用户明确要求在信息不足时继续，请基于默认假设生成，但必须在输出开头列出这些默认假设。

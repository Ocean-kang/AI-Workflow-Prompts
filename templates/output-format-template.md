# Output Format Template

## 1. Basic Information

| Item            | Content |
| --------------- | ------- |
| Task Name       |         |
| Input Files     |         |
| Output Language |         |
| Analysis Depth  |         |
| User Focus      |         |

---

## 2. Task Understanding

说明本次任务要解决什么问题。

需要回答：

* 用户希望完成什么？
* 输入文件是什么？
* 输出结果应该用于什么场景？
* 用户重点关注哪些问题？

---

## 3. Source Content Summary

总结输入文件中的核心内容。

要求：

* 只总结与任务相关的信息；
* 不要大段复述原文；
* 不要添加输入文件中不存在的事实；
* 如果信息不完整，需要明确指出。

---

## 4. Core Analysis

根据具体 workflow 进行分析。

可以包括：

* 背景问题
* 核心方法
* 关键模块
* 实验结果
* 代码结构
* 风险问题
* 优缺点
* 可行性
* 改进建议

具体内容由对应 workflow 决定。

---

## 5. Key Findings

用条目总结最重要的发现。

格式建议：

| Finding | Evidence | Interpretation |
| ------- | -------- | -------------- |
|         |          |                |

---

## 6. Limitations and Uncertainties

说明当前分析中的限制。

包括：

* 输入文件缺失
* 论文未说明
* 代码未实现
* 实验结果不足
* 需要进一步验证的地方

---

## 7. Conclusion

给出明确结论。

要求：

* 不要空泛；
* 不要过度肯定；
* 需要说明结论依据；
* 如果结论不确定，需要说明原因。

---

## 8. Next Steps

给出后续建议。

建议应当：

* 具体；
* 可执行；
* 有优先级；
* 与用户目标相关。

示例：

| Priority | Next Step | Purpose |
| -------- | --------- | ------- |
| High     |           |         |
| Medium   |           |         |
| Low      |           |         |

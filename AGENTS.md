# AGENTS.md

## 角色

你正在维护一个可复用的 AI 工作流和 Prompt 模板仓库。你的职责是帮助整理、改进和扩展 Markdown 形式的工作流文档，让它们能稳定服务于科研、代码开发、论文阅读、资料检索和其他 AI 辅助任务。

## 仓库工作方式

- 在进行结构性修改前，优先阅读 `SPEC.md`。
- 如果存在当前任务文件，优先阅读 `local-tasks/` 下对应的任务说明。
- 面向仓库的说明文档默认使用中文；只有当既有文件明显是英文语境时才保持英文。
- 修改应聚焦在用户要求的工作流、Prompt 或模板上。
- 保持仓库以 Markdown 文档为主的风格。
- 除非任务明确要求，不要引入应用代码、构建工具或生成型资源。

## 编辑准则

- Prompt 模板应具体、结构化、可复用。
- 优先包含使用场景、所需输入、执行步骤、输出格式、约束条件和失败处理。
- 变量占位符保持清晰风格，例如 `{USER_GOAL}` 或 `{OUTPUT_FORMAT}`。
- 避免无法验证的空泛指令。
- 修改已有 Prompt 时，应保留原本的目标用户、任务范围和输出意图。
- 不为了解释而过度扩写；模板应方便用户直接复制、改写和执行。
- 新增重要任务 Prompt 时，除非用户明确只需要单一版本，否则应同时生成通用执行版（Direct Mode）和交互询问版（Interactive Mode / Ask-First Mode）。

## 本地任务文件

- 具体任务的计划、执行记录和临时上下文应放在 `local-tasks/` 下。
- `local-tasks/` 中的文件只用于本地，不提交到 Git。
- 任务文件建议使用 `task001.md`、`task002.md` 这样的命名方式。
- 跨设备共享的长期规则应写入 `AGENTS.md` 或 `SPEC.md`，不要只写在本地任务文件中。

## 验证要求

完成修改前，应检查：

- Markdown 标题、列表和代码块能够正常阅读。
- 输出格式对 AI 助手或人工用户来说可以直接使用。
- 本地任务文件不会出现在 Git 的待提交列表中。
- 共享文档保持通用，不只服务某一次临时任务。
